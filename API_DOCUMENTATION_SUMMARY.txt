================================================================================
API ENDPOINTS
================================================================================

POST /api/posts/create
  - Create a new job post
  - Requires: Authorization Bearer token
  - Returns: Success/error response with post URL

GET /api/categories
  - Get all job categories
  - Returns: List of categories with IDs

GET /api/states
  - Get all states/union territories
  - Returns: List of states with IDs

COMMON USE CASES
================================================================================

1. Post a Single Job
   - Prepare job data
   - Send POST request to /api/posts/create
   - Get response with job URL

2. Bulk Post Multiple Jobs
   - Prepare jobs.json with all jobs
   - Run bulk posting script (Python or Bash)
   - Monitor progress and errors

3. Integrate with Your System
   - Get API token from environment
   - Fetch categories and states
   - Transform your data to API format
   - Post jobs programmatically
   - Handle errors and retries

4. Post Different Content Types
   - Change type field (job, admit_card, result, etc.)
   - Adjust content accordingly
   - Send POST request

================================================================================
BEST PRACTICES
================================================================================

1. Use HTTPS only - Always use https:// URLs
2. Validate data - Check all required fields before posting
3. Rate limiting - Add delays between bulk posts (1-2 seconds)
4. Error handling - Implement retry logic for failed requests
5. Keep token secure - Never share your API token
6. Test first - Test with one post before bulk posting
7. Use meaningful content - Provide complete job details
8. Monitor responses - Check for errors and handle them
9. Rotate token regularly - Change API token every 90 days
10. Document your integration - Keep records of what you're posting
