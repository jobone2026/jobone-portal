DOMAIN-BASED STATE FILTERING IMPLEMENTATION
============================================

OBJECTIVE:
- karnatakajob.online shows ONLY Karnataka jobs
- jobone.in shows ALL jobs from ALL states
- jobone.in/state/karnataka continues to work as before

FILES CREATED:
--------------
1. app/Http/Middleware/DomainStateFilter.php
   - New middleware to detect domain and apply state filtering
   - Reads DOMAIN_STATE_MAP from .env
   - Sets config values for controllers to use

2. DOMAIN_STATE_FILTERING.md
   - Complete documentation of the feature
   - Configuration examples
   - Troubleshooting guide

3. KARNATAKA_DOMAIN_SETUP.md
   - Quick deployment guide
   - Step-by-step server setup
   - DNS and Nginx configuration

4. setup-domain-filtering.sh
   - Interactive setup script
   - Helps configure .env file

5. verify-domain-filter.php
   - Verification script
   - Checks configuration and database

FILES MODIFIED:
---------------
1. bootstrap/app.php
   - Registered DomainStateFilter middleware
   - Applied to all web routes

2. app/Http/Controllers/HomeController.php
   - Added domain state filtering to homepage sections
   - Separate cache keys for filtered content

3. app/Http/Controllers/PostController.php
   - Added filtering to index() method
   - Added filtering to loadMore() method
   - Supports all post types (jobs, admit cards, results, etc.)

4. app/Http/Controllers/SearchController.php
   - Added filtering to search results
   - Added filtering to autocomplete

5. app/Http/Controllers/CategoryController.php
   - Added filtering to category pages

6. .env
   - Added DOMAIN_STATE_MAP configuration
   - Example: DOMAIN_STATE_MAP=karnatakajob.online:karnataka

HOW IT WORKS:
-------------
1. User visits karnatakajob.online
2. DomainStateFilter middleware intercepts request
3. Checks domain against DOMAIN_STATE_MAP in .env
4. Finds match: karnatakajob.online → karnataka
5. Looks up Karnataka state in database
6. Sets config values:
   - app.domain_state_id
   - app.domain_state_slug
   - app.domain_state_name
7. All controllers check config('app.domain_state_id')
8. If set, add WHERE state_id = X to queries
9. Only Karnataka posts are returned

CACHE STRATEGY:
---------------
- Normal cache key: home_sections
- Filtered cache key: home_sections_state_{state_id}
- Prevents cache conflicts between domains
- Each domain has independent cache

API ENDPOINTS:
--------------
- API is NOT affected by domain filtering
- External systems use state_id parameter
- Example: /api/posts?type=job&state_id=1

EXTENSIBILITY:
--------------
To add more state-specific domains:
  DOMAIN_STATE_MAP=karnatakajob.online:karnataka,maharashtrajobs.com:maharashtra

No code changes needed - just update .env and restart PHP-FPM
