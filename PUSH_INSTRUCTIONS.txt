═══════════════════════════════════════════════════════════════════════════
  SOLUTION 1: Use Personal Access Token (EASIEST)
═══════════════════════════════════════════════════════════════════════════

1. Login to GitHub as 'jobone2026'

2. Go to: https://github.com/settings/tokens

3. Click "Generate new token (classic)"

4. Fill in:
   - Note: "JobOne Deployment"
   - Expiration: 90 days (or No expiration)
   - Select scopes: ✓ repo (CRITICAL - This gives full control of repositories)
                    ✓ Make sure the 'repo' checkbox is CHECKED!

5. Click "Generate token"

6. COPY THE TOKEN (you won't see it again!)
   - It will start with "ghp_" followed by 40 characters
   - Example: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

7. In your terminal, run these commands:

   # First, update the remote URL with your new token
   git remote set-url origin https://YOUR_NEW_TOKEN@github.com/jobone2026/jobone.git
   
   # Then push
   git push -u origin main

   Replace YOUR_NEW_TOKEN with the actual token you copied.

Example:
# Update remote with new token
git remote set-url origin https://ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx@github.com/jobone2026/jobone.git

# Push to GitHub
git push -u origin main

═══════════════════════════════════════════════════════════════════════════
  SOLUTION 2: Clear Windows Credentials
═══════════════════════════════════════════════════════════════════════════

1. Open "Credential Manager" in Windows:
   - Press Windows key
   - Type "Credential Manager"
   - Click on it

2. Click "Windows Credentials"

3. Find entries for "git:https://github.com"

4. Click on each one and click "Remove"

5. Try pushing again:
   git push -u origin main

6. When prompted:
   - Username: jobone2026
   - Password: [Your Personal Access Token from Solution 1]

═══════════════════════════════════════════════════════════════════════════
  SOLUTION 3: Use Git Credential Manager
═══════════════════════════════════════════════════════════════════════════

1. Remove cached credentials:
   git credential-manager-core erase https://github.com

2. Or unset credential helper:
   git config --global --unset credential.helper

3. Push again:
   git push -u origin main

4. Login with jobone2026 credentials when prompted
