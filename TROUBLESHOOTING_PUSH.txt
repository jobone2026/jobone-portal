This error means the Personal Access Token doesn't have the correct permissions
OR the repository doesn't exist yet.

═══════════════════════════════════════════════════════════════════════════
  SOLUTION 1: VERIFY REPOSITORY EXISTS
═══════════════════════════════════════════════════════════════════════════

1. Login to GitHub as 'jobone2026'

2. Check if repository exists:
   https://github.com/jobone2026/jobone

3. If it DOESN'T exist, create it:
   - Go to: https://github.com/new
   - Repository name: jobone
   - Description: Government Job Portal - JobOne.in
   - Visibility: Private or Public (your choice)
   - ⚠️  DO NOT check "Initialize with README"
   - ⚠️  DO NOT add .gitignore or license
   - Click "Create repository"

═══════════════════════════════════════════════════════════════════════════
  SOLUTION 2: VERIFY TOKEN HAS 'REPO' SCOPE
═══════════════════════════════════════════════════════════════════════════

1. Go to: https://github.com/settings/tokens

2. Find your token in the list (it will show partial characters)

3. Check what scopes it has:
   - If it doesn't show 'repo' scope, you need to create a NEW token

4. To create a NEW token with correct permissions:
   - Click "Generate new token" → "Generate new token (classic)"
   - Note: "JobOne Deployment"
   - Expiration: 90 days or No expiration
   - ✅ CHECK THE 'repo' SCOPE (this is critical!)
   - Click "Generate token"
   - Copy the new token (starts with ghp_)

5. Update remote with new token:
   git remote set-url origin https://NEW_TOKEN@github.com/jobone2026/jobone.git
   git push -u origin main

═══════════════════════════════════════════════════════════════════════════
  SOLUTION 3: CHECK TOKEN OWNERSHIP
═══════════════════════════════════════════════════════════════════════════

Make sure the token was created by the 'jobone2026' account:

1. Login to GitHub as 'jobone2026'
2. Go to: https://github.com/settings/tokens
3. Verify the token is listed there
4. If not, you're using a token from a different account

Common issues:

❌ "Repository not found"
   → Create the repository first at https://github.com/new

❌ "Permission denied"
   → Token doesn't have 'repo' scope or belongs to wrong account

❌ "Authentication failed"
   → Token is expired or invalid, generate a new one

❌ "Remote already exists"
   → This is fine, just update it with: git remote set-url origin ...
